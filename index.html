Why these three

The Head Cloud CoE mandate spans governance, financial stewardship, and competitive strategy simultaneously. Each simulator isolates one axis of that mandate so the trade-offs — and their second-order effects on cost, compliance, and time-to-market — become visible and arguable rather than asserted.

Alignment to UBS Group cloud strategy

  • Azure-first, multi-cloud capable — models default to Microsoft Azure as the primary hyperscaler for UBS Card Center AG, consistent with UBS Group's enterprise landing zone strategy, with AWS/private cloud treated as governed exceptions rather than parallel default paths.
  • Shared responsibility & group governance inheritance — the CoE is modelled as extending UBS Group's cloud control framework (architecture standards, security baselines, FinOps reporting) into the regulated card-issuing entity, not building a parallel one.
  • FINMA / PCI-DSS / DORA as hard gates — compliance is modelled as a release gate, not a checkbox: under-resourcing security or architecture capability visibly delays go-live readiness.
  • Product Operating Model & Platform Engineering — team and velocity assumptions reflect the agile-at-scale, self-service platform approach described in the accompanying vision document, rather than a centralised ticket-driven IT model.
